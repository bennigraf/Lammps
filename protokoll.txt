DRAFT!

Module: Speicherbaustein -> melden sich mit Funktionen
Bei mehreren Funktionen: ein Paket pro Funktion

RF-Setup:
2mbit
2byte crc
dynamic payload width
no autoack
5byte addr width
channel 127 for discovery! changeable via configuration later...

address 0 for registration with central node
# central node waits on addr 0 for incoming registration requests with guid from certain module
# recv packet: addr(0), data[(info)0x00, (func)]
# answers with: addr(0), data[(info)0x00, (func)0x01, (nbytes)guid, (5bytes)addr]
## (on unsuccessful registration due to packet loss module repeats registration. Registration with double guid drops the first one on the central node.)
# => module goes into rx mode


RF-Packet towards central node:
byte 0: info (see below)
byte 1: command
10:		registration-request from module -- (n bytes)guid, (byte)func1, (byte)func2, (byte)funcn...



RF-Paket towards module:
byte 0: info
7: startpacket of multipacket-transaction
6: packet is part of multipacket-transaction
# 4-0: payload-width (do we need this with dpl where the rf knows how much data is coming in?)

byte 1: command
0:		reset/reboot
1-9: 	reserved for configuraiton
	1: set rx address for certain guid
10: 	raw data
20: 	dim
21:		multi-dim
30:		rgb
31:		multi-rgb
40:		sound/synth

byte 2-31: payload
